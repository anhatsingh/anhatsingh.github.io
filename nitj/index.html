Source Code (using git CLI)

Git Bash is an application that provides Git command line experience on the Operating System. It is a command-line shell for enabling git with the command line in the system. A shell is a terminal application used to interface with an operating system through written commands. Git Bash is a package that installs Bash, some common bash utilities, and Git on a Windows operating system. In Git Bash the user interacts with the repository and git elements through the commands.

Download and install the git command-line-interface (cli) based on your OS:

  • Windows: Download the windows binary file from https://git-scm.com/downloads and follow the on-screen instructions to install
  • Ubuntu: Run the following command: 
    $ sudo apt install git
    Ensure that the packages are installed correctly by running the following: 
    $ git --version

Run the following commands in shell (CMD/Windows PowerShell/bash) after installation:

$ git config --global user.name "John Doe"
$ git config --global user.email johndoe@example.com

Note

Change John Doe and johndoe@example.com with your Github name and email

The source code of the website can be cloned from anhatsingh's github repository using the following command in the working directory:

$ git clone https://github.com/anhatsingh/nitj_website_2021

Note

Git CLI may require github email and password for the first time.

Node.js and npm

Node.js is an open-source, cross-platform, back-end JavaScript runtime environment that runs on the V8 engine and executes JavaScript code outside a web browser. Node.js lets developers use JavaScript to write command line tools and for server-side scriptingâ€”running scripts server-side to produce dynamic web page content before the page is sent to the user's web browser. Consequently, Node.js represents a "JavaScript everywhere" paradigm,[6] unifying web-application development around a single programming language, rather than different languages for server-side and client-side scripts.

  • Windows: Download the windows binary file from nodejs.org and follow the on-screen instructions to install
  • Ubuntu: run the following commands to install nodejs and npm on ubuntu 
    $ sudo apt install nodejs
$ sudo apt install npm
    Ensure that the packages are installed correctly by running the following: 
    $ node --version
$ npm --version

Angular.js CLI

The Angular CLI is a command-line interface tool that you use to initialize, develop, scaffold, and maintain Angular applications directly from a command shell.

Install the CLI using the npm package manager: 

$ npm install -g @angular/cli

After installing Angular.js, open Shell in the frontend directory and run the following to get the build tools for angular:

$ npm install --save-dev @angular-devkit/build-angular

PostgreSQL & pgAdmin

PostgreSQL is a powerful, open source object-relational database system with over 30 years of active development that has earned it a strong reputation for reliability, feature robustness, and performance.

  • Windows: Download the windows binary file from postgresql.org and follow the on-screen instructions to install
  • Ubuntu: run the following commands to install: (There are many methods but i recommend the following:)

    Installing PostgreSQL

    1. Install Synaptic Package Manager, a powerful tool to install packages in Ubuntu.

      $ sudo add-apt-repository universe /* You can use any repository you like, this is the one I found on the net. */
$ sudo apt update
$ sudo apt install synaptic
    2. Go to Database tab, and right-click on PostgreSQL 12 and click on Mark for Installation button.
    3. Click on Apply to complete installation.
    4. Ensure the Package is installed using the following commands:
      5. $ sudo systemctl is-active postgresql
$ sudo systemctl is-enabled postgresql
$ sudo systemctl status postgresql
    5. Also, confirm that the Postgresql server is ready to accept connections from clients as follows: 
      $ sudo pg_isready

You should see output like this:
/var/run/postgresql:5432 - connection is ready

    Creating Database in PostgreSQL

    1. To create a new database in PostgreSQL, you need to access the PostgreSQL database shell (psql) program. First, switch to the postgres system user account and run the psql command as follows:

      $ sudo su - postgres
$ psql
    2. Now create a new database and a user using the following commands.

      3. $ postgres=# CREATE USER johndoe WITH PASSWORD 'my_secret_password';
$ postgres=# CREATE DATABASE testDb;
$ postgres=# GRANT ALL PRIVILEGES ON DATABASE testDb to johndoe;
$ postgres=# \q

    Installing pgAdmin

    1. pgAdmin4 is not available in the Ubuntu repositories. We need to install it from the pgAdmin4 APT repository. Start by setting up the repository. Add the public key for the repository and create the repository configuration file.

      $ sudo apt install curl
$ curl https://www.pgadmin.org/static/packages_pgadmin_org.pub | sudo apt-key add
$ sudo sh -c 'echo "deb https://ftp.postgresql.org/pub/pgadmin/pgadmin4/apt/$(lsb_release -cs) pgadmin4 main" > /etc/apt/sources.list.d/pgadmin4.list && apt update'

    2. Then install pgAdmin4:

      $ sudo apt install pgadmin4

    3. Set username and password for pgAdmin4 by typing the following command and following the on-screen instructions:

      $ sudo /usr/pgadmin4/bin/setup-web.sh

    4. To access the pgAdmin4 web application interface, open a web browser, and use the following address to navigate:

      http://SERVER_IP/pgadmin4

    5. After a successful login, you will be land in the pgAdmin4 web application dashboard. To connect to a server, click on Add New Server as highlighted in the following screenshot.

    6. Next, enter the connection in General settings (Name, Server group, and a comment). Then click Connections as highlighted in the following screenshot.

    7. Next, enter the PostgreSQL database server hostname/address, Port number (leave 5432 to use default), select the Maintenance database (which should be postgres), enter the database username and password.

    8. If the database access credentials are OK and the server-client authentication configuration is too, pgAdmin4 should successfully connect to the database server.

    All Done!
